SOCKR

A type-safe WebSocket messaging framework with authentication, presence tracking, typing indicators, and React hooks.

npm install sockr-server sockr-client
Get Started → GitHub

Built for Real-time Apps

Everything you need for person-to-person WebSocket communication, from authentication to typing indicators.

⚡

Real-time Messaging

Send and receive messages instantly between authenticated users with delivery confirmation and error handling.

🔒

Authentication

Token-based authentication with a simple handler pattern. Validate tokens from JWT, database lookups, or any custom source.

🟢

Presence Tracking

Know who's online in real-time. Automatic broadcasts on connect/disconnect with batch status queries.

✏️

Typing Indicators

Show when users are typing with automatic timeout handling. Built into both server and client SDKs.

⚛

React Hooks

7 purpose-built hooks for React apps: useSocket, useMessages, useSendMessage, usePresence, useTypingIndicator, and more.

🛠

TypeScript First

Full type safety across the entire stack. Shared types ensure server and client stay in sync at compile time.

How It Works

Three packages that work together. Shared types keep the server and client in sync.

Server
sockr-server
Plugin-based WebSocket server with auth, presence, and messaging
← → imports
Shared
sockr-shared

Client

Packages

Install only what you need. Each package is published independently to npm.

sockr-server

v1.1.0

Plugin-based WebSocket server built on Socket.IO. Authentication, presence, messaging, typing indicators, and a custom plugin system.

View docs →

sockr-client

v1.0.0

Client SDK with React hooks for real-time messaging. SocketClient class works with or without React. Auto-reconnection with exponential backoff.

View docs →

sockr-shared

v1.0.0

Shared TypeScript type definitions. Zero runtime dependencies. SocketEvent enum, EventPayloads, and all configuration interfaces.
